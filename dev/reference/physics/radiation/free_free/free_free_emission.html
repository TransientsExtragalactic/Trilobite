Free-Free Emission Tools#

Hint

See Free-Free Radiation Theory for a detailed overview of the underlying physics of free-free (bremsstrahlung) radiation.

The trilobite.radiation.free_free module provides a complete set of tools for computing thermal free-free emission and absorption in ionized astrophysical plasmas. The module is organized into three sub-modules:

  • core — monochromatic emissivity and absorption coefficients.

  • gaunt_factor — Gaunt factor evaluation (analytic approximation and tabulated interpolators).

  • absorption — free-free optical depth integration over a variety of CSM density profiles.

Each sub-module follows Trilobite’s standard two-level API pattern: a private, log-space backend optimized for numerical performance and a public, unit-aware interface that accepts and returns astropy.units.Quantity objects.

Emissivity and Absorption Coefficients#

The core module implements the monochromatic thermal free-free emissivity \(j_\nu\) and absorption coefficient \(\alpha_\nu\) together with their Rayleigh–Jeans and Wien limiting forms.

The Two-Level API#

All computations are performed in two layers:

  • Private backends (_log_ff_*) operate entirely in natural-logarithm CGS space. All six inputs (log_nu, log_n_e, log_n_i, log_Z, log_T, g_ff) are expected in natural log CGS form, and the functions return the natural log of the result. Working in log-space provides numerical stability over the many decades of dynamic range that arise in astrophysical plasmas and eliminates unit overhead in performance-critical loops.

  • Public wrappers (compute_ff_*) perform unit coercion via ensure_in_units(), delegate to the appropriate backend, and return Quantity objects with appropriate CGS units.

Important

The private backends accept plain floats assumed to be in natural-log CGS. Passing un-logged values or quantities in non-CGS units to the private functions will produce silently incorrect results. Unless you are building a performance-critical pipeline, use the public API.

Emissivity#

The thermal free-free spectral emissivity follows Rybicki and Lightman[1], Eq. 5.14:

\[j_\nu = C_{\rm ff}\, Z^2\, n_e\, n_i\, T^{-1/2}\, e^{-h\nu / k_B T}\, g_{\rm ff},\]

where \(C_{\rm ff} = 6.8 \times 10^{-38}\) CGS.

Trilobite implements this via compute_ff_emissivity():

The public API accepts astropy.units.Quantity inputs and returns the emissivity as a Quantity in erg s-1 cm-3 Hz-1 sr-1:

import numpy as np
from astropy import units as u
from trilobite.radiation.free_free.core import compute_ff_emissivity

j = compute_ff_emissivity(
    nu=1e9 * u.Hz,
    n_e=1e3 * u.cm**-3,
    n_i=1e3 * u.cm**-3,
    Z=1,
    T=1e4 * u.K,
)
print(j.to(u.erg / u.s / u.cm**3 / u.Hz / u.sr))

Plain floats are also accepted and interpreted as Hz, cm-3, and K respectively—the Gaunt factor g_ff is always dimensionless:

j = compute_ff_emissivity(nu=1e9, n_e=1e3, n_i=1e3, Z=1, T=1e4)

The private backend _log_ff_emissivity() accepts natural logarithms of all arguments:

import numpy as np
from trilobite.radiation.free_free.core import _log_ff_emissivity

log_j = _log_ff_emissivity(
    log_nu=np.log(1e9),   # ln(ν / Hz)
    log_n_e=np.log(1e3),  # ln(n_e / cm⁻³)
    log_n_i=np.log(1e3),
    log_Z=np.log(1.0),
    log_T=np.log(1e4),    # ln(T / K)
    g_ff=5.0,
)
j = np.exp(log_j)  # erg s⁻¹ cm⁻³ Hz⁻¹ sr⁻¹

The Rayleigh–Jeans emissivity—valid when \(h\nu \ll k_BT\)—drops the exponential factor and is frequency-independent, scaling as \(T^{-1/2}\). It is implemented via compute_ff_RJ_emissivity() (high-level) and _log_ff_RJ_emissivity() (log-space backend).

Note

The log_nu argument is accepted by _log_ff_RJ_emissivity for interface consistency but does not contribute to the result. The output is broadcast to the shape of log_nu so that the function behaves identically to the full emissivity when used in vectorized expressions.

Absorption Coefficient#

The thermal free-free absorption coefficient is (Rybicki and Lightman[1], Eq. 5.19):

\[\alpha_\nu = C_\alpha\, Z^2\, n_e\, n_i\, T^{-1/2}\, \nu^{-3}\, \bigl(1 - e^{-h\nu / k_B T}\bigr)\, g_{\rm ff},\]

where \(C_\alpha = 3.7 \times 10^{8}\) CGS.

This is implemented via compute_ff_absorption():

import numpy as np
from astropy import units as u
from trilobite.radiation.free_free.core import compute_ff_absorption

alpha = compute_ff_absorption(
    nu=1e9 * u.Hz,
    n_e=1e4 * u.cm**-3,
    n_i=1e4 * u.cm**-3,
    Z=1,
    T=1e4 * u.K,
)
print(alpha.to(u.cm**-1))

The result can be integrated along a line of sight to obtain the optical depth. For a uniform slab of thickness \(L\):

nu_arr = np.geomspace(1e8, 1e12, 300) * u.Hz
alpha_arr = compute_ff_absorption(
    nu=nu_arr, n_e=1e5 * u.cm**-3, n_i=1e5 * u.cm**-3, Z=1, T=1e4 * u.K
)
tau = (alpha_arr * 1e17 * u.cm).decompose()

The log-space backend _log_ff_absorption() evaluates \(\log(1 - e^{-h\nu/k_BT})\) via numpy.log1p to avoid catastrophic cancellation at low frequencies where \(h\nu/k_BT \ll 1\):

import numpy as np
from trilobite.radiation.free_free.core import _log_ff_absorption

log_alpha = _log_ff_absorption(
    log_nu=np.log(1e9),
    log_n_e=np.log(1e4),
    log_n_i=np.log(1e4),
    log_Z=np.log(1.0),
    log_T=np.log(1e4),
    g_ff=5.0,
)

The Rayleigh–Jeans approximation—valid when \(h\nu \ll k_BT\)—replaces \((1-e^{-h\nu/k_BT}) \to h\nu/k_BT\), yielding \(\alpha_\nu^{\rm RJ} \propto \nu^{-2} T^{-3/2}\). This is the standard radio bremsstrahlung opacity scaling. It is implemented via compute_ff_RJ_absorption() and its log-space backend.

Gaunt Factor Evaluation#

The gaunt_factor module provides two approaches to evaluating the free-free Gaunt factor \(g_{\rm ff}(Z, T, \nu)\).

The Gaunt factor depends on two dimensionless collision parameters:

\[u \equiv \frac{h\nu}{k_B T}, \qquad \gamma^2 \equiv \frac{Z^2 R_y}{k_B T},\]

where \(R_y = 13.6\,{\rm eV}\) is the Rydberg energy. Trilobite uses these parameters internally to map physical inputs \((Z, T, \nu)\) onto the table axes.

Analytic Approximation#

A fast closed-form approximation due to Draine[2] (Eq. 10.9) is provided by gaunt_ff_draine(). It accepts unit-aware or plain-float inputs and is suitable for quick estimates in the radio through infrared regime:

from trilobite.radiation.free_free.gaunt_factor import gaunt_ff_draine

# Scalar
gff = gaunt_ff_draine(Z=1, T=1e4, nu=1e10)

# With astropy units
from astropy import units as u
gff = gaunt_ff_draine(Z=1, T=1e6 * u.K, nu=5.0 * u.GHz)

Note

The Draine approximation is accurate to a few percent in the RJ regime but can deviate more significantly at high frequencies or extreme temperatures. For production-quality calculations, the tabulated interpolators described below are preferred.

Tabulated Interpolators#

Trilobite ships two pre-computed Gaunt factor tables from van Hoof et al.[3] and provides a class hierarchy for convenient, accurate interpolation.

Class

Table dimensions

When to use

NonRelativisticGauntFactorInterpolator

2D (\(\log_{10} u\), \(\log_{10}\gamma^2\))

Default; appropriate for most radio and thermal-plasma applications

RelativisticGauntFactorInterpolator

3D (\(Z\), \(\log_{10} u\), \(\log_{10}\gamma^2\))

High-temperature (\(T \gtrsim 10^7\) K) or high-\(Z\) plasma

Both classes share the same interface: they are callable with \((Z, T, \nu)\) and support the Python in operator to test whether a point lies within the table boundaries.

The NonRelativisticGauntFactorInterpolator wraps a 2D scipy.interpolate.RegularGridInterpolator() over the non-relativistic van Hoof (2014) table. The ionic charge \(Z\) enters only through the coordinate transform to \(\log_{10}\gamma^2\) and is not a table axis.

The default instance can be obtained via the factory function get_default_gaunt_interpolator():

import numpy as np
from trilobite.radiation.free_free.gaunt_factor import (
    get_default_gaunt_interpolator,
)

nr = get_default_gaunt_interpolator()

# Scalar evaluation
gff = nr(Z=1, T=1e4, nu=1e10)
print(gff)           # ≈ 4.46

# Array evaluation
T_arr = np.geomspace(1e4, 1e8, 50)
gff_arr = nr(Z=1, T=T_arr, nu=1e10)

# Domain check
print((1, 1e4, 1e10) in nr)  # True

The RelativisticGauntFactorInterpolator wraps a 3D scipy.interpolate.RegularGridInterpolator(). Z is a first-class table axis, allowing the interpolator to capture the explicit Z-dependence of relativistic corrections beyond what is encoded in \(\log_{10}\gamma^2\) alone.

The default relativistic instance can be obtained via get_default_relativistic_gaunt_interpolator():

import numpy as np
from trilobite.radiation.free_free.gaunt_factor import (
    get_default_relativistic_gaunt_interpolator,
)

r = get_default_relativistic_gaunt_interpolator()

# Scalar evaluation
gff = r(Z=1, T=1e4, nu=1e10)
print(gff)           # ≈ 4.71

# Domain check
print((1, 1e4, 1e10) in r)  # True

Class Hierarchy#

Both interpolators share a common abstract base class, GauntFactorInterpolatorBase, which defines the public interface and provides the HDF5 loading helper. Users implementing custom Gaunt factor tables can subclass this base class to integrate them into the Trilobite API.

from trilobite.radiation.free_free.gaunt_factor import (
    GauntFactorInterpolatorBase,
    NonRelativisticGauntFactorInterpolator,
    RelativisticGauntFactorInterpolator,
    get_default_gaunt_interpolator,
    get_default_relativistic_gaunt_interpolator,
)

nr = get_default_gaunt_interpolator()
r  = get_default_relativistic_gaunt_interpolator()

assert isinstance(nr, GauntFactorInterpolatorBase)
assert isinstance(r,  GauntFactorInterpolatorBase)

print(nr.description)  # Non-relativistic free-free Gaunt factors (...)
print(r.description)   # Relativistic free-free Gaunt factors (...)

Free-Free Optical Depth#

The absorption module provides routines to integrate \(\alpha_\nu(r)\) along a line of sight through an ionized CSM for five physically motivated density profiles. For each profile type there is both an exact variant (using the full \((1 - e^{-h\nu/k_BT})\) factor) and a Rayleigh–Jeans variant (using the approximation \(\propto \nu^{-2} T^{-3/2}\)). All public functions accept and return unit-bearing quantities; the quadrature-based variants accept CGS-only callables for maximum flexibility.

Important

For the quadrature-based profiles (_from_quadrature variants), the callable arguments must return values in CGS units without any astropy unit overhead. This is the only place in the public API where plain CGS callables are required; all other parameters remain unit-aware.

Profile Overview#

Function pair

Description

compute_ff_optical_depth_from_quadrature() / compute_ff_RJ_optical_depth_from_quadrature()

Adaptive quadrature over arbitrary user-supplied CGS callables for \(n_e(r)\), \(n_i(r)\), \(T(r)\). Maximum flexibility for non-analytic or tabulated CSM models.

compute_ff_optical_depth_from_arrays() / compute_ff_RJ_optical_depth_from_arrays()

Trapezoidal integration over pre-computed (\(r\), \(\alpha_\nu\)) or (\(r\), \(n_e\), \(n_i\), \(T\)) grids. Ideal for post-processing hydrodynamic simulation outputs.

compute_ff_optical_depth_wind() / compute_ff_RJ_optical_depth_wind()

Analytic integral for a steady stellar wind with \(\rho \propto r^{-2}\) (constant mass-loss rate \(\dot{M}\), wind velocity \(v_w\)). Gives \(\tau_{\rm ff} \propto r^{-3}\).

compute_ff_optical_depth_shell() / compute_ff_RJ_optical_depth_shell()

Analytic integral for a uniform-density shell between radii \(r\) and \(r_{\rm max}\). Returns \(\tau_{\rm ff} = \alpha_\nu (r_{\rm max} - r)\).

compute_ff_optical_depth_powerlaw() / compute_ff_RJ_optical_depth_powerlaw()

Analytic integral for a generalized power-law density profile \(\rho \propto r^{-p}\) between \(r\) and \(r_{\rm max}\). Encompasses the wind (\(p = 2\)) and uniform (\(p = 0\)) cases.

Usage Examples#

Compute the free-free optical depth through a CSM described by arbitrary number density and temperature profiles, using adaptive quadrature from \(r\) to \(r_{\rm max}\):

import numpy as np
from astropy import units as u
from trilobite.radiation.free_free.absorption import (
    compute_ff_optical_depth_from_quadrature,
)

# Define CGS callables (no astropy units inside)
def n_e(r):
    return 1e3 * (r / 1e15) ** -2  # cm⁻³, wind profile

def n_i(r):
    return n_e(r)

def T(r):
    return 1e4  # Kelvin, uniform temperature

tau = compute_ff_optical_depth_from_quadrature(
    nu=1e9 * u.Hz,
    Z=1.0,
    n_e_func=n_e,
    n_i_func=n_i,
    T_func=T,
    r=1e15 * u.cm,
    r_max=2e15 * u.cm,
    g_ff=5.0,
)
print(tau)  # dimensionless optical depth

For a steady stellar wind with mass-loss rate \(\dot{M}\) and wind velocity \(v_w\), use the analytic wind integral:

from astropy import units as u
from trilobite.radiation.free_free.absorption import (
    compute_ff_optical_depth_wind,
    compute_ff_RJ_optical_depth_wind,
)

# Wind parameters
mdot = 1e-5 * u.Msun / u.yr
v_w  = 10   * u.km / u.s

tau_exact = compute_ff_optical_depth_wind(
    nu=1e9 * u.Hz,
    Z=1.0,
    mdot=mdot,
    v_wind=v_w,
    T=1e4 * u.K,
    r=1e15 * u.cm,
    g_ff=5.0,
)

tau_rj = compute_ff_RJ_optical_depth_wind(
    nu=1e9 * u.Hz,
    Z=1.0,
    mdot=mdot,
    v_wind=v_w,
    T=1e4 * u.K,
    r=1e15 * u.cm,
    g_ff=5.0,
)

For a uniform-density shell, the integral reduces to \(\tau = \alpha_\nu (r_{\rm max} - r)\):

from astropy import units as u
from trilobite.radiation.free_free.absorption import (
    compute_ff_optical_depth_shell,
)

tau = compute_ff_optical_depth_shell(
    nu=1e9 * u.Hz,
    Z=1.0,
    n_e=1e3 * u.cm**-3,
    n_i=1e3 * u.cm**-3,
    T=1e4 * u.K,
    r=1e15 * u.cm,
    r_max=2e15 * u.cm,
    g_ff=5.0,
)

For a generalized \(\rho \propto r^{-p}\) profile, the integral is computed analytically. The power-law exponent is \(p = 0\) (uniform) or \(p = 2\) (wind) as special cases:

from astropy import units as u
from trilobite.radiation.free_free.absorption import (
    compute_ff_optical_depth_powerlaw,
)

# General p = 1.5 profile
tau = compute_ff_optical_depth_powerlaw(
    nu=1e9 * u.Hz,
    Z=1.0,
    n_e=1e3 * u.cm**-3,
    n_i=1e3 * u.cm**-3,
    T=1e4 * u.K,
    r=1e15 * u.cm,
    r_max=2e15 * u.cm,
    p=1.5,
    g_ff=5.0,
)

Warning

For power-law profiles with \(p > 0.5\) (i.e. \(k \equiv 1-2p < 0\)), the optical depth integral converges as \(r_{\rm max} \to \infty\). Setting r_max=np.inf is valid and will return the asymptotic total optical depth in this case. For \(p < 0.5\) the integral diverges at \(r_{\rm max} \to \infty\) and a finite upper limit must be provided.
