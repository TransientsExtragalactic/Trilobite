One-Zone Accretion Disk Models#

The one-zone disk module provides time-dependent accretion disk evolution in a compact, inference-friendly package. The disk is reduced to a two-component ODE system (mass \(M_D\) and angular momentum \(J_D\)), integrated by a compiled Cython explicit-Euler kernel that evaluates all thermodynamic and structural quantities at every step.

This document covers everything a user needs to run simulations with existing models and everything a developer needs to add new thermodynamic closures. For the underlying physics see One-Zone Accretion Disk Models: Theory.

Quick Start#

The minimal workflow is four steps: instantiate a disk model, call solve(), and read the result.

from astropy import constants as const
from astropy import units as u
from triceratops.dynamics.accretion.one_zone import GasPressureElectronScatteringDisk

# 1. Create the disk model.
disk = GasPressureElectronScatteringDisk(mu=0.62)

# 2. Solve the evolution.
result = disk.solve(
    initial_conditions={
        "M_D_0": 0.1 * const.M_sun,
        "J_D_0": 1.5e49 * u.g * u.cm**2 / u.s,
    },
    runtime_parameters={
        "M_BH": 3.0 * const.M_sun,
        "R_in": 3.0e6 * u.cm,
        "alpha": 0.1,
    },
    t_span=(1.0e6 * u.s, 5.0e9 * u.s),
    max_steps=50_000,
)

# 3. Access the results.
data = result.data
print(data["T_c"][[0, -1]])     # central temperature at start and end
print(data["R_D"][[0, -1]].to(u.cm))   # outer radius in cm

Hint

All parameter values and time-span endpoints accept either plain floats (in the base CGS unit for that parameter) or Quantity objects — unit conversion is applied automatically by the solver.

Creating a Disk Model#

Each disk class is instantiated with context parameters — physical constants that are fixed for the lifetime of a model instance and do not change between solves.

Parameter

Description

Default

mu

Mean molecular weight \(\mu\) (dimensionless). Appropriate values are \(\approx 0.6\) for a fully-ionised solar-composition plasma or \(\approx 0.5\) for a pure hydrogen plasma.

0.6

 
disk = GasPressureElectronScatteringDisk(mu=0.62)

Parameter

Description

Default

mu

Mean molecular weight \(\mu\) (dimensionless).

0.6

 
disk = FullPressureElectronScatteringDisk(mu=0.62)

The Metzger+08 geometry constants \(A = 1.62\), \(B = 1.33\), and \(F_0 = 1.6\) are compiled into the Cython layer and are accessible as read-only class attributes disk._A, disk._B, and disk._F.

Setting Up Initial Conditions#

The solver requires the initial disk mass \(M_{D,0}\) and angular momentum \(J_{D,0}\). The simplest approach is to set them manually:

initial_conditions = {
    "M_D_0": 0.1 * const.M_sun,
    "J_D_0": 1.5e49 * u.g * u.cm**2 / u.s,
}

In many practical situations it is more natural to specify the disk outer radius \(R_{D,0}\) rather than the angular momentum. The helper method generate_initial_conditions() uses the Metzger+08 kinematic constraint

\[J_D = \xi\, M_D\, \sqrt{G M_{\rm BH} R_D}, \qquad \xi \equiv B/A,\]

to infer the missing variable from any two of \(M_{D,0}\), \(J_{D,0}\), and \(R_{D,0}\).

ic = disk.generate_initial_conditions(
    M_BH=3.0 * const.M_sun,
    M_D_0=0.1 * const.M_sun,
    R_D_0=3.0e13 * u.cm,     # supply R_D_0 instead of J_D_0
)
# ic = {"M_D_0": ..., "J_D_0": ...}

result = disk.solve(
    initial_conditions=ic,
    runtime_parameters={"M_BH": 3.0 * const.M_sun, "R_in": 3.0e6 * u.cm, "alpha": 0.1},
    t_span=(1.0e6 * u.s, 5.0e9 * u.s),
)

Exactly two of the three disk state variables must be provided.

Running the Solver#

solve() is the primary entry point.

result = disk.solve(
    initial_conditions=ic,
    runtime_parameters=run_params,
    t_span=(1.0e6 * u.s, 5.0e9 * u.s),
    max_steps=100_000,
    fail_fast=True,
)

Runtime parameters are the per-solve physics inputs:

Type

Default

Description

int

100_000

Hard limit on integration steps. The solver terminates early if t_end is reached first.

If True, re-raise the integrator RuntimeError on failure. If False, issue a warning and return the (possibly empty) result.

Integrator Error Codes#

If the Cython integrator encounters an unrecoverable failure it raises a RuntimeError with one of the following messages:

Code

Name

Meaning

-1

CLOSURE / FUNC_ERROR

The root-finding residual returned NaN or Inf (unphysical disk state).

-2

CLOSURE / EXPAND_FAIL

Bracket expansion could not find a sign change — no thermal equilibrium exists for this set of disk parameters.

-3

CLOSURE / NO_BRACKET

Both bracket endpoints had the same sign after expansion.

-4

CLOSURE / MAX_ITER

Brent’s method did not converge within the iteration limit.

-10

DERIVATIVE_FAIL

The ODE derivative function returned a non-zero status.

-20

WRITER_FAIL

The result-writer function returned a non-zero status.

Hint

EXPAND_FAIL typically indicates extreme parameters (very high surface density, very high \(\alpha\)) where the energy balance has no physical equilibrium. Reducing \(\alpha\) or the initial disk mass usually resolves the issue.

Accessing Results#

solve() returns a OneZoneAccretionResult dataclass.

The data Property#

The primary interface to all output fields is the data property, which returns a dictionary of unit-bearing Quantity arrays indexed by field name:

data = result.data

print(data["T_c"])          # central temperature, shape (N,)  [K]
print(data["R_D"].to(u.au)) # outer disk radius in AU
print(data["mdot"])         # accretion rate, shape (N,)  [g/s]

Subscript notation is also supported:

T_c = result["T_c"]   # equivalent to result.data["T_c"]

State-Variable Shortcuts#

Frequently needed state variables are also available as direct properties:

Snapshots: Initial and Final State#

initial_state and final_state return dictionaries containing every field evaluated at the first and last timestep respectively:

t0 = result.initial_state
tf = result.final_state

print(f"Initial T_c  = {t0['T_c']:.3e}")
print(f"Final M_D    = {tf['M_D'].to(u.Msun):.4f}")

Diagnostic Utilities#

Before committing to a long parameter-grid sweep it is useful to sanity-check the initial conditions.

Single-Step Solve#

solve_initial_state() runs exactly one integration step and returns the corresponding OneZoneAccretionResult:

result0 = disk.solve_initial_state(
    initial_conditions=ic,
    runtime_parameters=run_params,
)
print(result0["T_c"])

Physical Self-Consistency Checks#

test_initial_state() runs a set of physical consistency checks on the initial disk state and returns a report dictionary:

report = disk.test_initial_state(
    initial_conditions=ic,
    runtime_parameters=run_params,
)
for name, result in report.items():
    status = "PASS" if result["passed"] else "FAIL"
    print(f"  [{status}] {name}: {result['note']}")

The following checks are performed:

Check name

Condition

Diagnostic hint

finite

All output fields are finite

Indicates unphysical disk state (extreme parameters)

T_c_positive

\(T_c > 0\)

Should always pass if finite passes

tau_thick

\(\tau > 1\) (optically thick)

Required for the diffusion approximation to hold

H_over_R

\(H/R \ll 1\) (geometrically thin)

Thin-disk assumption; values \(\gtrsim 0.3\) indicate a fat disk

R_D_gt_R_in

\(R_D > R_{\rm in}\)

The disk must extend outside the inner truncation radius

Time Interpolation#

The explicit-Euler integrator outputs values at adaptively spaced timesteps governed by the local viscous timescale. To evaluate a field at an arbitrary time use the interpolation utilities.

Getting a Spline Interpolator#

get_field_interpolator() returns a InterpolatedUnivariateSpline over the integration time axis:

interp = result.get_field_interpolator("T_c")       # cubic spline (k=3)
interp_lin = result.get_field_interpolator("mdot", k=1)  # linear spline

Evaluating at New Times#

interpolate_field() handles unit conversion and reattaches the correct Quantity units to the output:

t_eval = np.geomspace(1e6, 5e9, 500) * u.s
T_c_interp = result.interpolate_field("T_c", t_eval)
mdot_interp = result.interpolate_field("mdot", t_eval)

A pre-computed interpolator can be reused by passing it as the interpolator keyword, avoiding redundant spline construction in loops:

interp = result.get_field_interpolator("T_c")
for t in t_grid:
    T = result.interpolate_field("T_c", t, interpolator=interp)

Parameter Grid Sweeps#

solve_parameter_grid() evaluates the disk model over a full Cartesian product of parameter values:

results = disk.solve_parameter_grid(
    initial_conditions_grid={
        "M_D_0": [0.05 * const.M_sun, 0.1 * const.M_sun, 0.2 * const.M_sun],
    },
    runtime_parameters_grid={
        "M_BH": [3.0 * const.M_sun, 10.0 * const.M_sun],
        "R_in": [3.0e6 * u.cm],
        "alpha": [0.05, 0.1, 0.3],
    },
    t_span=(1.0e6 * u.s, 5.0e9 * u.s),
    max_steps=50_000,
    fail_fast=False,    # continue on failure so one bad run doesn't abort the grid
)

The return value is a dictionary keyed by tuples of (parameter_name, value) pairs (sorted alphabetically) to OneZoneAccretionResult objects. A tqdm progress bar is shown automatically when the project configuration enables progress bars.

Note

Each combination is run sequentially. For large grids consider wrapping solve() calls in a concurrent.futures.ProcessPoolExecutor for parallelism.

Physical Observables#

The result object provides convenience methods for deriving common astrophysical observables.

Accretion Luminosity#

compute_luminosity() evaluates the gravitational accretion luminosity \(L = G M_{\rm BH} \dot{M} / R_{\rm in}\) at a specified time:

L = result.compute_luminosity(1.0e8 * u.s)
print(L.to(u.Lsun))

An existing \(\dot{M}\) spline interpolator can be reused with the interpolator keyword.

Spectral (Specific) Luminosity#

compute_specific_luminosity() computes the disk spectral luminosity by integrating a multi-colour blackbody spectrum over the disk face:

\[L_\nu = 4\pi^2 \int_{R_{\rm in}}^{R_D} B_\nu\!\left(T(R)\right)\, R\, dR, \qquad T(R) = T_{\rm eff}(R_D) \left(\frac{R_D}{R}\right)^{3/4}.\]
import astropy.units as u

L_nu = result.compute_specific_luminosity(
    time=1.0e8 * u.s,
    frequency=3.0e14 * u.Hz,   # optical V-band
    n_radii=500,
)
print(L_nu.to(u.erg / (u.s * u.Hz)))

Disk Effective Temperature Profile#

compute_thin_disk_effective_temperature() evaluates the multi-colour temperature profile at a given radius and time:

\[T_{\rm eff}(R, t) = T_{\rm eff}(R_D, t)\,\left(\frac{R_D}{R}\right)^{3/4}.\]
radii = np.geomspace(3e6, 3e13, 200) * u.cm
T_profile = result.compute_thin_disk_effective_temperature(
    radius=radii,
    time=1.0e8 * u.s,
)

df = result.to_dataframe()   # requires pandas

Note

to_dataframe() requires pandas, which is not a mandatory dependency of Triceratops. If not installed, a descriptive ImportError is raised directing you to pip install pandas.

Serialisation Round-Trip#

Disk model instances (not results) can be serialised to a plain dict and reconstructed:

spec = disk.to_spec_dict()
import json
print(json.dumps(spec, indent=2))
# {"target": "...core:GasPressureElectronScatteringDisk", "mu": 0.62}

disk2 = GasPressureElectronScatteringDisk.from_spec_dict(spec)

Developer Guide: Adding a New Closure#

Adding a new one-zone disk closure requires three steps: writing the Cython extension, declaring the Python model class, and registering the tests.

Step 1 — Write the Cython closure#

Each closure is a subclass of OneZoneClosure defined in a .pyx file. The subclass must install three C function pointers in its __cinit__:

  • _closure_fn — evaluates the energy balance, solves for \(T_c\), and fills a ClosureResult struct.

  • _derivative_fn — evaluates \((\dot{M}_D, \dot{J}_D)\) and computes the adaptive timestep.

  • _writer_fn — writes all output quantities to the result array at the current column index.

The function signatures are declared in triceratops.dynamics.accretion.one_zone._integrator (.pxd); import them with cimport. The ClosureResult struct carries the standard set of thermodynamic outputs:

cdef struct ClosureResult:
    double dM_dt       # g s^-1  (negative: drain)
    double dJ_dt       # g cm^2 s^-2
    double t_visc      # s   (warm-start seed for next step)
    double log_T_eff   # ln(K)
    double log_T_c     # ln(K)
    double log_tau     # ln(dimensionless)
    double log_cs      # ln(cm s^-1)
    double log_nu      # ln(cm^2 s^-1)
    double log_q_visc  # ln(erg cm^-2 s^-1)

The writer function must also fill any extra rows specific to your closure. Set n_result_fields in __cinit__ to the total number of rows you intend to write.

Step 2 — Write the Python model class#

Subclass OneZoneAccretionDiskBase and declare the four required class-level dictionaries:

class MyClosure(OneZoneAccretionDiskBase):

    # Geometry constants (must match the compiled Cython layer)
    _A: float = 1.62
    _B: float = 1.33
    _F: float = 1.6

    # Parameters fixed at construction time
    CONTEXT_PARAMETERS = {
        "mu": {"description": "Mean molecular weight", "base_units": None, "default": 0.6},
        # add your own...
    }

    # Parameters passed to each solve() call
    RUNTIME_PARAMETERS = {
        "M_BH":  {"description": "BH mass",            "base_units": "g",  "default": None, "log_transform": True},
        "R_in":  {"description": "Inner radius",        "base_units": "cm", "default": None, "log_transform": True},
        "alpha": {"description": "Shakura-Sunyaev alpha","base_units": None,"default": None, "log_transform": False},
    }

    # Initial conditions consumed by solve()
    INITIAL_CONDITIONS = {
        "M_D_0": {"description": "Initial disk mass",     "base_units": "g",         "default": None, "log_transform": True},
        "J_D_0": {"description": "Initial angular momentum","base_units": "g*cm**2/s","default": None, "log_transform": True},
    }

    # All output fields
    RESULT_FIELDS = {
        "R_D":   {"description": "Disk outer radius",  "units": "cm"},
        "T_c":   {"description": "Midplane temperature","units": "K"},
        # ... add your own fields
    }

    # Maps RESULT_FIELDS keys → row index in the Cython result array
    # (None = field not directly in result_array)
    CYTHON_FIELD_MAP = {
        "R_D": 4,
        "T_c": 8,
        # ...
    }

    def _build_cython_closure(self):
        from ._my_closure import MyCythonClosure
        return MyCythonClosure()

    def _pack_cython_parameters(self, run_params):
        return np.array([
            np.exp(run_params["log_M_BH"]),
            np.exp(run_params["log_R_in"]),
            run_params["alpha"],
            self._context_parameters["mu"],
            # ... any additional parameters your closure needs
        ], dtype=np.float64)

    def to_spec_dict(self):
        return {"target": "...:MyClosure", "mu": self._context_parameters["mu"]}

    @classmethod
    def from_spec_dict(cls, data):
        return cls(**{k: v for k, v in data.items() if k != "target"})

Important

The metaclass _OneZoneMeta validates the four declaration dicts at class-definition time. If a required key is missing or has the wrong type, you will see a descriptive TypeError immediately — not buried inside a solver traceback.

Step 3 — Add tests#

The test module tests/test_dynamics/test_one_zone_disk_base.py provides BaseTestOneZoneDisk, a shared pytest base class that runs the full suite of physical consistency and round-trip checks for any concrete disk model. To add tests for your new closure:

# tests/test_dynamics/test_my_closure.py
from test_one_zone_disk_base import BaseTestOneZoneDisk
from triceratops.dynamics.accretion.one_zone import MyClosure

class TestMyClosure(BaseTestOneZoneDisk):
    MODEL = MyClosure
    DISK_KWARGS = {"mu": 0.62}

    M_BH  = 3.0 * const.M_sun
    M_D_0 = 0.1 * const.M_sun
    R_IN  = 3.0e6 * u.cm
    R_D_0 = 3.0e13 * u.cm
    ALPHA = 0.1

    # Add any closure-specific tests below ...
    def test_my_specific_physics(self):
        ...

Run the full test suite with:

pytest tests/test_dynamics/ -v
